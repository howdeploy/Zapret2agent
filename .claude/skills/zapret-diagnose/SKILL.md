# Диагностика системы и zapret

**Когда использовать:** перед любым действием с zapret, при запросе пользователя «диагностика» / «что с системой» / «проверь» / «статус».

---

## Как запустить диагностику

```bash
bash .claude/skills/zapret-diagnose/scripts/detect-state.sh
```

Прочитать JSON-вывод. НЕ парсить вручную — JSON самодостаточен. Не спрашивать подтверждение перед запуском: detect-state.sh не требует root, это read-only проверка.

---

## Таблица интерпретации JSON-полей

| Поле | Значение | Реакция агента |
|------|----------|----------------|
| `dns_hijack_suspected: true` | DNS может быть отравлен | **ПЕРВЫМ ДЕЛОМ** предупредить: «Похоже, твой провайдер подменяет DNS-ответы. Это значит что некоторые сайты перенаправляются на страницу блокировки ещё на уровне DNS, до того как zapret может помочь. Дополнительно к zapret нужно решить DNS-проблему отдельным способом.» НЕ продолжать диагностику пока не объяснил. |
| `vpn_active: true` | VPN/прокси включён | Сообщить: «Обнаружен {vpn_client} ({vpn_protocol_family}). Exit IP: {exit_ip}. Если zapret не работает — попробуй отключить VPN/прокси.» |
| `vpn_client` | Конкретный клиент | Показать в таблице диагностики: throne, nekoray, hiddify, v2raya, amneziavpn, clash, sing-box, wireguard, openvpn. Для unknown-tun: «Обнаружен неизвестный TUN-клиент». |
| `vpn_protocol_family` | Семейство протокола | Для xray-family: «Используется VLESS/VMess/Trojan-подобный протокол». Для amnezia-wg: «Используется AmneziaWG». Для unknown: «Протокол не определён». |
| `exit_ip` | Внешний IP | Показать в таблице диагностики. Если отличается от IP без VPN — VPN работает корректно. Если unknown — «Не удалось определить exit IP (нет доступа к ifconfig.me)». |
| `firewall_backend: iptables-nft` | Современный backend | Использовать nftables-совместимые команды. При показе правил — `nft list ruleset` вместо `iptables -L`. |
| `firewall_backend: iptables-legacy` | Старый backend | Использовать стандартные `iptables -L -n -v` команды. |
| `firewall_backend: nftables` | Чистый nftables | Использовать только `nft` команды. |
| `firewall_backend: unknown` | Не определён | Предупредить: «Не удалось определить firewall backend. Это может быть нестандартная конфигурация.» |
| `zapret_installed: false` | Не установлен | Предложить: «Zapret не установлен. Хочешь установить?» |
| `zapret_running: false` + `zapret_installed: true` | Установлен но не запущен | Предложить: «Zapret установлен но не запущен. Запустить?» |
| `zapret_config_exists: false` + `zapret_installed: true` | Нет конфига | Предупредить: «Zapret установлен, но файл конфигурации отсутствует. Нужна настройка.» |
| `init_system: openrc` | Не systemd | Использовать `rc-service` вместо `systemctl`. safe-mode.sh использует fallback без systemd-run. |
| `dns_ok: false` | DNS не работает | «DNS-резолвинг не работает. Проверь подключение к интернету.» |
| `os: unknown` | ОС не определена | Сообщить: «Не удалось определить операционную систему. Убедись что запускаешь агент на Linux.» |
| `kernel` | Версия ядра | Показывать в таблице диагностики. Нет особых реакций если < 5.x (только информация). |

## Формат вывода диагностики (CLAUDE.md)

Показывать результат диагностики в виде таблицы:

```
| Параметр           | Значение              | Статус         |
|--------------------|----------------------|----------------|
| ОС                 | Manjaro Linux         | OK             |
| Ядро               | 6.12.68-1-MANJARO     | OK             |
| Init-система       | systemd               | OK             |
| Firewall           | iptables-nft          | OK             |
| VPN-клиент         | нет                   | OK             |
| VPN-протокол       | —                     | OK             |
| Exit IP            | —                     | OK             |
| Zapret установлен  | нет                   | ❗ проблема    |
| Zapret запущен     | нет                   | ❗ проблема    |
| DNS-резолвинг      | работает              | OK             |
| DNS-отравление     | не обнаружено         | OK             |
| Интерфейсы         | eth0, wlan0           | OK             |
```

Статусы: **OK** / **⚠️ внимание** / **❗ проблема**

После таблицы — сразу предлагать рекомендации на основе найденного.

## Ограничения DNS-проверки

- DNS-расхождение помечается как `dns_hijack_suspected` (**подозрение**, не подтверждение).
- CDN-домены (youtube.com) могут давать разные IP от разных resolver даже без отравления — поэтому скрипт проверяет 3 домена и требует 2 из 3 расхождений.
- Агент должен объяснить пользователю: «Это подозрение, не 100% факт. CDN-сервисы иногда дают разные IP из разных точек интернета.»

## Быстрая проверка здоровья (health-check.sh)

**Когда использовать:** при запросе «всё ли ок?» / «проверь здоровье» / «quick check» / «health check» — когда нужна быстрая сводка вместо полной диагностики.

```bash
bash .claude/skills/zapret-diagnose/scripts/health-check.sh
```

Скрипт вызывает detect-state.sh + manage-service.sh status и агрегирует результаты в единый JSON.

### Интерпретация JSON

| Поле | Значения | Реакция агента |
|------|----------|----------------|
| `overall` | `"ok"` | Всё в порядке, сообщить пользователю |
| `overall` | `"warning"` | Показать `issues[]` и объяснить каждый пункт |
| `overall` | `"critical"` | **Срочно** показать `issues[]`, предложить действия по исправлению |

### Когда critical

- `zapret установлен, но сервис не запущен` — предложить: «Zapret установлен, но не работает. Запустить?»
- `подозрение на перехват DNS-трафика провайдером` — предупредить о DNS hijack (как в таблице выше)

### Когда warning

- `VPN активен` — предупредить о возможном конфликте
- `DNS-резолвер не отвечает` — предложить проверить подключение к интернету

### Отличие от detect-state.sh

| | detect-state.sh | health-check.sh |
|---|---|---|
| Цель | Полная JSON-диагностика системы | Быстрая сводка здоровья |
| Когда | Перед любым действием с zapret | При запросе «всё ли ок?» |
| Вывод | 16 полей — полное состояние | overall + issues — сводка |
| Вызывает | Ничего | detect-state.sh + manage-service.sh |

---

## Важные правила

- detect-state.sh **не требует root**. Не спрашивать подтверждение перед запуском.
- Скрипт запускается **перед любым действием** с zapret — никогда не предполагай состояние системы без него.
- При `dns_hijack_suspected: true` — предупреждать **первым**, до любых рекомендаций по zapret.
- При `vpn_active: true` — предупреждать о конфликте, но **не блокировать** продолжение диагностики.
