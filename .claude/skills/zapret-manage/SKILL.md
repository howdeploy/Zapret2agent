# Управление сервисом и списками (zapret-manage)

**Когда использовать:** пользователь говорит «статус zapret», «запусти/останови zapret», «перезапусти zapret», «добавь домен», «удали домен», «покажи список», «что в списке», «управление zapret». Или агент при повторном запуске показывает краткий статус.

---

## Секция 1: Предварительная проверка (обязательно)

Перед любой операцией управления запустить:

```bash
bash .claude/skills/zapret-diagnose/scripts/detect-state.sh
```

Проверить JSON-вывод:

| Поле | Что делать |
|------|------------|
| `zapret_installed: false` | «zapret2 не установлен. Сначала установи его — скажи мне "установить zapret".» СТОП. |
| `zapret_installed: true` | Продолжить. |
| `init_system` | Запомнить значение (`systemd` / `openrc`) — контекст для объяснений. |
| `zapret_running: true` | Запомнить — влияет на сообщение о применении доменов. |
| `zapret_running: false` | Запомнить — при добавлении домена сообщить что он начнёт работать после запуска. |

---

## Секция 2: Статус сервиса (MGMT-01)

Запустить:

```bash
bash .claude/skills/zapret-manage/scripts/manage-service.sh status
```

Скрипт возвращает JSON:
```json
{"status":"active","init_system":"systemd","detail":"...вывод systemctl..."}
```

**Интерпретация по значению `status`:**

| status | Ответ агента |
|--------|--------------|
| `active` | «Сервис zapret2 работает нормально.» |
| `inactive` | «Сервис zapret2 остановлен. Запустить? (да/нет)» |
| `failed` | «Сервис zapret2 упал с ошибкой. Вот что пошло не так:» + показать `detail` в `<details>` блоке |
| `activating` | «Сервис zapret2 запускается, подожди несколько секунд...» — перепроверить через 3 секунды. |
| `unknown` | «Не удалось определить статус сервиса. Скорее всего init_system не определён.» + показать `detail` |

**Формат вывода агента (таблица):**

```
Статус zapret2:

Параметр     | Значение
-------------|----------
Состояние    | Работает / Остановлен / Ошибка
Init-система | systemd / openrc
```

## Секция 3: Управление сервисом (MGMT-02)

**Определить действие из запроса пользователя:**
- «запусти», «включи», «старт» → `start`
- «останови», «выключи», «стоп» → `stop`
- «перезапусти», «рестарт» → `restart`

**ОБЯЗАТЕЛЬНО запросить подтверждение перед выполнением (протокол безопасности из CLAUDE.md).** Без явного «да» — не выполнять.

Шаблон подтверждения:
```
Вот что я хочу сделать:
  sudo systemctl {action} zapret2   (или: sudo rc-service zapret2 {action})

{Объяснение действия}. {Предупреждение если restart}.
Выполнить? (да/нет)
```

Объяснение по действию:
- `start` → «Это запустит DPI-обход трафика.»
- `stop` → «Это остановит DPI-обход. Сеть продолжит работать, но блокировки вернутся.»
- `restart` → «Это перезапустит DPI-обход. На секунду сеть может моргнуть.»

После подтверждения пользователя запустить:
```bash
bash .claude/skills/zapret-manage/scripts/manage-service.sh {action}
```

**Интерпретация вывода:**

Скрипт выводит сначала `ACTION_OK` или `ACTION_FAILED`, затем свежий JSON статуса.

| Вывод | Ответ агента |
|-------|--------------|
| `ACTION_OK` | «Готово! {Статус из JSON}.» + таблица статуса как в Секции 2 |
| `ACTION_FAILED` | «Что-то пошло не так. Вот детали:» + показать `detail` из JSON |

**После start/restart:** «Проверь доступ к нужному сайту — если не работает, скажи мне.»

## Секция 4: Добавление домена (MGMT-03)

Получить домен (или несколько) от пользователя.

Для каждого домена запустить:
```bash
bash .claude/skills/zapret-manage/scripts/manage-hostlist.sh add "{domain}"
```

**Интерпретация вывода:**

| Вывод | Ответ агента |
|-------|--------------|
| `NORMALIZED=example.com` + `DOMAIN_ADDED` | «Добавил "example.com" (убрал www. или https://).» + пояснение ниже |
| Только `DOMAIN_ADDED` (без NORMALIZED) | «Добавил "{domain}" в список.» + пояснение ниже |
| `DOMAIN_ALREADY_EXISTS` | «Домен "{domain}" уже есть в списке. Ничего не изменилось.» |
| `ERROR_INVALID_DOMAIN` | «"{domain}" — не похоже на домен. Примеры: youtube.com, rutracker.org. Попробуй ещё раз.» |
| `ERROR_MISSING_DOMAIN` | «Не указан домен. Напиши какой домен добавить.» |

**Пояснение после DOMAIN_ADDED** (выбрать в зависимости от `zapret_running` из Секции 1):
- Если zapret2 работает: «Zapret2 применит его немедленно — перезапуск не нужен.»
- Если zapret2 остановлен: «Домен добавлен в список. Он начнёт действовать после запуска сервиса.»

**Несколько доменов:** если пользователь дал несколько через запятую или пробел — обработать каждый отдельно, сообщить результат по каждому.

**Нормализация — объяснить пользователю:**
Если был `NORMALIZED=example.com` при вводе `www.example.com` → «Добавил "example.com" — убрал "www." потому что zapret2 покрывает субдомены автоматически.»

## Секция 5: Удаление домена (MGMT-04)

Получить домен от пользователя.

Запустить:
```bash
bash .claude/skills/zapret-manage/scripts/manage-hostlist.sh remove "{domain}"
```

**Интерпретация вывода:**

| Вывод | Ответ агента |
|-------|--------------|
| `DOMAIN_REMOVED` + `HUP_SENT` | «Удалил "{domain}" из списка. Изменения применены немедленно.» |
| `DOMAIN_REMOVED` + `HUP_SKIPPED` | «Удалил "{domain}" из списка. Чтобы применить — перезапусти zapret2.» |
| `DOMAIN_NOT_FOUND` | «Домен "{domain}" не найден в вашем списке. Возможно, он записан в другом формате.» + предложить показать список (Секция 6) |
| `HOSTLIST_NOT_FOUND` | «Файл списка не существует — значит, вы ещё не добавляли домены вручную.» |
| `ERROR_MISSING_DOMAIN` | «Не указан домен. Напиши какой домен удалить.» |

**При `DOMAIN_NOT_FOUND`:** если домен похож на субдомен (содержит точку в начале или выглядит как `cdn.example.com`) — подсказать: «Попробуй указать основной домен: example.com. Показать весь список?»

## Секция 6: Просмотр списков (MGMT-05)

Запустить:
```bash
bash .claude/skills/zapret-manage/scripts/manage-hostlist.sh list
```

Скрипт возвращает JSON:
```json
{
  "user_count": 3,
  "auto_count": 47,
  "user_domains": ["youtube.com", "rutracker.org", "discord.com"],
  "auto_sample": ["example.com", "...первые 20 доменов из auto..."]
}
```

**Формат ответа агента:**

```
Ваш список (zapret-hosts-user.txt): {user_count} доменов — добавлены вручную

{Если user_count > 0:}
  1. youtube.com
  2. rutracker.org
  3. discord.com

{Если user_count == 0:}
  Вы пока не добавляли домены вручную.

---

Автосписок (zapret-hosts-auto.txt): {auto_count} доменов — zapret2 добавил сам

Автосписок работает так: когда вы пытаетесь открыть сайт и соединение
обрывается или идут повторные попытки — это признак DPI-блокировки.
zapret2 замечает это и автоматически запоминает домен для обхода.

Это нормальное поведение, трогать не нужно.
```

**Если `auto_count > 20`:** «Показываю первые 20 из {auto_count}.» + вывод `auto_sample`.

**Если `auto_count == 0`:** Не показывать секцию автосписка, или: «Автосписок пока пустой — он заполнится по мере работы zapret2.»

## Секция 7: Мерж seed-списка (SEED-03)

**Когда предлагать мерж:**
- После завершения установки zapret2 (Phase 2 workflow)
- Если пользователь спрашивает про seed-список, начальный список, стартовый набор
- Если в результате `list` у пользователя 0 доменов и zapret2 установлен

**Workflow:**

1. Объяснить пользователю:
   «В репозитории есть готовый список из ~50 заблокированных доменов — YouTube, Discord, Telegram, соцсети, СМИ, dev-ресурсы. Хочешь добавить их в свой список?»

2. Если пользователь согласен — сначала показать что будет (dry-run):
   ```bash
   bash .claude/skills/zapret-manage/scripts/merge-seed.sh --dry-run
   ```

3. Интерпретировать вывод dry-run:
   - `BEFORE_COUNT=N` → «Сейчас у тебя N доменов в списке»
   - `AFTER_COUNT=N` → «После мержа будет N доменов»
   - `ADDED_COUNT=N` → «Добавится N новых доменов (дубликаты автоматически исключены)»

4. Запросить подтверждение:
   ```
   Вот что я хочу сделать:
     Добавить {ADDED_COUNT} доменов из seed-списка в твой hostlist.
     Текущие домены сохранятся, дубликатов не будет.
   Выполнить? (да/нет)
   ```

5. После подтверждения — выполнить мерж:
   ```bash
   bash .claude/skills/zapret-manage/scripts/merge-seed.sh
   ```

6. Интерпретировать вывод:
   | Вывод | Ответ агента |
   |-------|--------------|
   | `MERGE_COMPLETE` + `HUP_SENT` | «Готово! Было {BEFORE_COUNT}, стало {AFTER_COUNT} доменов (+{ADDED_COUNT} новых). Изменения применены немедленно.» |
   | `MERGE_COMPLETE` + `HUP_SKIPPED` | «Готово! Было {BEFORE_COUNT}, стало {AFTER_COUNT}. Запусти zapret2 чтобы применить.» |
   | `ERROR_SEED_NOT_FOUND` | «Файл seed-списка не найден. Убедись что репозиторий не повреждён.» |
   | `ERROR_ZAPRET_NOT_INSTALLED` | «zapret2 не установлен. Сначала установи его.» |

7. Если `ADDED_COUNT=0`:
   «Все домены из seed-списка уже есть в твоём списке. Ничего не изменилось.»

## Секция 8: Anti-patterns — что НЕЛЬЗЯ делать

**1. НЕ перезапускать сервис после изменений hostlist**

После `add` или `remove` скрипт уже отправил SIGHUP демонам nfqws2. Этого достаточно для немедленного применения. Restart сбросит autohostlist в памяти и разорвёт активные соединения. Никогда не предлагать restart после операций с hostlist.

**2. НЕ редактировать zapret-hosts-auto.txt**

Это автоматический список, управляется nfqws2. Если пользователь просит удалить оттуда домен — объяснить: «Этот домен в автосписке, который zapret2 ведёт сам. Если его удалить вручную — при следующем обнаружении блокировки он вернётся автоматически. Хочешь, добавим домен в ваш ручной список?»

**3. НЕ пропускать подтверждение перед start/stop/restart**

Протокол безопасности из CLAUDE.md. Даже если пользователь говорит «просто перезапусти» — показать команду и спросить «Выполнить? (да/нет)».

**4. НЕ добавлять домен с www. префиксом вручную**

zapret2 покрывает субдомены автоматически. Скрипт нормализует `www.example.com` → `example.com`. Агент объясняет пользователю что произошло: «Убрал "www." — zapret2 обходит блокировку для всего домена целиком, включая субдомены.»

**5. НЕ использовать `--hostlist=/path` напрямую**

Только `<HOSTLIST>` placeholder в NFQWS2_OPT (правило из Phase 3). Это касается конфигурации, а не hostlist — не смешивать понятия.

**6. НЕ предполагать init_system**

Скрипт manage-service.sh определяет его сам. Агент не должен хардкодить команды типа `sudo systemctl` — только вызывать скрипт.

**7. НЕ редактировать seed-list.txt**

seed-list.txt — статический файл в репозитории. Если пользователь хочет добавить домен — использовать `manage-hostlist.sh add`, домен попадёт в пользовательский список (zapret-hosts-user.txt). Seed-список обновляется только через обновление репозитория (git pull).

## Секция 9: Скрипты

| Скрипт | Путь | Назначение |
|--------|------|------------|
| manage-service.sh | `.claude/skills/zapret-manage/scripts/manage-service.sh` | Статус + start/stop/restart с JSON-выводом |
| manage-hostlist.sh | `.claude/skills/zapret-manage/scripts/manage-hostlist.sh` | add/remove/list для пользовательского hostlist |
| merge-seed.sh | `.claude/skills/zapret-manage/scripts/merge-seed.sh` | Мерж seed-списка с пользовательским hostlist |
| detect-state.sh | `.claude/skills/zapret-diagnose/scripts/detect-state.sh` | Предварительная диагностика состояния системы |

**Аргументы manage-service.sh:**
- `status` → JSON статус
- `start` / `stop` / `restart` → `ACTION_OK` или `ACTION_FAILED` + JSON статус

**Аргументы manage-hostlist.sh:**
- `add <domain>` → `NORMALIZED=...` (если нормализован) + `DOMAIN_ADDED` / `DOMAIN_ALREADY_EXISTS` / `ERROR_INVALID_DOMAIN`
- `remove <domain>` → `DOMAIN_REMOVED` / `DOMAIN_NOT_FOUND` / `HOSTLIST_NOT_FOUND`
- `list` → JSON с user_count, auto_count, user_domains, auto_sample

**Аргументы merge-seed.sh:**
- (без аргументов) → мерж + `BEFORE_COUNT` + `AFTER_COUNT` + `ADDED_COUNT` + `MERGE_COMPLETE` + `HUP_SENT`/`HUP_SKIPPED`
- `--dry-run` → только подсчёт без записи: `DRY_RUN=true` + `BEFORE_COUNT` + `AFTER_COUNT` + `ADDED_COUNT`

## Секция 10: Полные workflow (6 сценариев)

### MGMT-01: Показать статус

```
1. Пользователь: «статус zapret»
   ↓
2. detect-state.sh → проверить zapret_installed
   → false: «не установлен» + СТОП
   → true: продолжить
   ↓
3. manage-service.sh status → JSON
   ↓
4. Интерпретировать status → показать таблицу пользователю
   → inactive: предложить запустить
   → failed: показать detail, помочь разобраться
```

### MGMT-02: Управление (start/stop/restart)

```
1. Пользователь: «запусти/останови/перезапусти zapret»
   ↓
2. detect-state.sh → проверить zapret_installed
   → false: СТОП
   ↓
3. Определить action из запроса пользователя
   ↓
4. Показать подтверждение с командой и объяснением
   → пользователь говорит «нет»: не выполнять
   ↓
5. manage-service.sh {action} → ACTION_OK или ACTION_FAILED
   ↓
6. Сообщить результат + текущий статус
   → start/restart: «Проверь доступ к нужному сайту»
```

### MGMT-03: Добавить домен

```
1. Пользователь: «добавь youtube.com»
   ↓
2. detect-state.sh → запомнить zapret_running
   ↓
3. manage-hostlist.sh add "youtube.com"
   ↓
4. Интерпретировать вывод:
   → NORMALIZED: объяснить нормализацию
   → DOMAIN_ADDED: подтвердить + указать когда начнёт работать
   → DOMAIN_ALREADY_EXISTS: сообщить что уже есть
   → ERROR_INVALID_DOMAIN: объяснить формат
   ↓
5. Если несколько доменов — повторить для каждого
```

### MGMT-04: Удалить домен

```
1. Пользователь: «удали youtube.com»
   ↓
2. manage-hostlist.sh remove "youtube.com"
   ↓
3. Интерпретировать вывод:
   → DOMAIN_REMOVED: подтвердить удаление
   → DOMAIN_NOT_FOUND: подсказать формат + предложить показать список
   → HOSTLIST_NOT_FOUND: объяснить что список пустой
```

### MGMT-05: Показать список

```
1. Пользователь: «покажи список», «что в списке», «какие домены»
   ↓
2. manage-hostlist.sh list → JSON
   ↓
3. Показать user_domains пронумерованным списком
   ↓
4. Объяснить автосписок (auto_count + auto_sample)
   → Акцент: это нормально, трогать не нужно
```

### SEED-03: Мерж seed-списка

```
1. Пользователь: «добавь seed-список», «начальный список», или 0 доменов после установки
   ↓
2. Объяснить что в seed-списке → предложить мерж
   ↓
3. merge-seed.sh --dry-run → показать счётчики (BEFORE_COUNT, AFTER_COUNT, ADDED_COUNT)
   ↓
4. Подтверждение пользователя
   → «нет»: не выполнять
   ↓
5. merge-seed.sh → MERGE_COMPLETE
   ↓
6. Сообщить результат с конкретными числами
   → ADDED_COUNT=0: «Все домены уже в списке»
   → HUP_SENT: «Изменения применены немедленно»
   → HUP_SKIPPED: «Запусти zapret2 чтобы применить»
```
